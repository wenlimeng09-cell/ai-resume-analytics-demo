# AI 简历优化助手 Demo + 用户行为数据分析与效果评估项目

## 项目定位

本项目是一个可本地演示的 MVP，用于验证 AI 简历优化助手的用户路径、建议采纳行为和产品效果评估方法。

项目基于模拟产品日志数据和小样本体验反馈，评估 AI 简历优化助手在简历优化流程中的转化效果、建议采纳效果和产品优化方向。

本项目不作为真实上线产品，不追踪真实投递和面试反馈，也不宣称真实提升面试率。

## MVP 功能范围

保留功能：

- 上传简历文本或文件；
- 粘贴目标岗位 JD；
- 基于规则模拟生成结构化简历优化建议；
- 展示“问题定位 + 修改建议 + 示例改写 + 匹配 JD 关键词”；
- 用户可以对每条建议点击“采纳 / 不采纳”；
- 记录用户行为日志；
- 支持导出优化建议报告；
- 增加数据分析看板，展示核心指标和图表。

暂不实现：

- 用户登录系统；
- 支付系统；
- 多用户权限系统；
- 真实投递追踪；
- 真实面试反馈追踪；
- 复杂推荐算法；
- 大规模真实用户实验。

## 数据结构

核心数据表包括：

- `users`：用户信息；
- `resume_tasks`：简历优化任务；
- `events`：行为日志；
- `suggestions`：优化建议；
- `feedback`：小样本体验反馈，可选。

新增 `resume_tasks` 的原因是：一个用户可能针对多个 JD 多次优化简历，只按 `user_id` 分析会导致分析口径混乱。

其中 `suggestions` 表新增 `original_expression`、`revised_expression` 和 `status` 字段，用于支持建议前后对比、待处理/已采纳/未采纳状态追踪；`events` 表新增 `suggestion_id` 和 `prompt_group`，用于追踪建议级行为和 Prompt A/B 分析。

## 技术实现说明

- 前端展示：Streamlit；
- 数据处理：Python / pandas；
- 建议生成：规则模拟，预留 API Provider；
- 数据存储：CSV 或 SQLite；
- 可视化：Plotly / Streamlit 图表；
- 分析方法：漏斗分析、分群分析、Prompt A/B 实验、建议质量评分分析。

## 项目边界说明

当前项目为 MVP Demo，主要验证产品流程和分析方法。用户行为数据主要为模拟产品日志，少量体验反馈可用于辅助验证，不代表真实上线后的商业结果；面试反馈率等长期结果指标未作为核心分析目标。

## 核心分析指标

漏斗路径：

访问产品 -> 上传简历 -> 上传 JD -> 生成建议 -> 采纳建议 -> 导出报告

北极星指标：

有效优化任务完成率 = 完成导出报告且至少采纳 1 条建议的任务数 / 生成建议的任务数。

重点分析：

- 各环节用户数、转化率和流失率；
- 建议曝光数、采纳数、采纳率和人均采纳建议数；
- 不同建议类型和不同简历模块的采纳率；
- Prompt A/B 两组的采纳率、导出率和平均建议质量评分；
- 建议质量评分与采纳率的关系；
- 应届生、实习生、转行用户的采纳率、导出率和人均优化任务数。

## 数据分析思路

1. 业务目标：提升用户完成有效简历优化任务的比例。
2. 核心指标：有效优化任务完成率。
3. 分析路径：漏斗定位问题 -> 分群拆解差异 -> A/B 验证 Prompt 效果 -> 建议质量评分分析采纳驱动因素 -> 输出产品优化优先级。
4. 问题诊断：围绕“为什么用户采纳了建议，但没有完成报告导出”分析 adopted_count、pending_count、match_score_lift、prompt_group、user_type 和 avg_quality_score 等因素。
5. 项目边界：当前分析基于模拟产品日志，用于展示数据分析框架和产品效果评估思路，不宣称真实提升面试率。

## 产品优化方向

- 将 AI 建议展示为“问题定位 + 修改建议 + 示例改写”三段式；
- 在每条建议旁展示匹配的 JD 关键词，增强可信度；
- 提供“一键采纳 / 复制改写文案 / 部分采纳”功能，降低采纳门槛；
- 针对转行用户增加“可迁移能力识别”和“目标岗位差距分析”；
- 增加“有用 / 无用”反馈按钮，持续优化建议生成逻辑。

基于当前模拟数据看板，可进一步收敛为：

- 将 AI 建议展示为“问题定位 + 修改建议 + 示例改写 + 前后对比”的结构；
- 增加 JD 关键词绑定解释，提升用户信任；
- 将导出报告按钮前置，降低最后一步流失；
- 针对转行用户增加可迁移能力识别和岗位差距分析；
- 增加“有用 / 无用”反馈按钮，持续优化建议生成逻辑。

## 产品优化优先级

| 优化方案 | 影响指标 | 预期收益 | 实现成本 | 优先级 | 数据依据 |
|---|---|---|---|---|---|
| 导出按钮前置 | 报告导出率、有效优化任务完成率 | 高 | 低 | P0 | 基于模拟产品日志，“采纳建议 -> 导出报告”流失最大 |
| 用户采纳建议后增加下一步引导 | 报告导出率 | 高 | 低 | P0 | 用户采纳建议后未必知道下一步动作 |
| 增加 JD 关键词绑定解释 | 建议采纳率 | 中高 | 中 | P1 | B 组 JD 匹配型 Prompt 采纳率更高 |
| 针对转行用户增加可迁移能力识别 | 转行用户导出率 | 中高 | 中高 | P1 | 转行用户能力映射难度更高 |
| 增加有用 / 无用反馈按钮 | 建议质量评分、后续采纳率 | 中 | 低 | P1 | 需要沉淀建议质量反馈数据 |
| 后续接入真实 API | 建议质量、用户信任 | 中 | 高 | P2 | 当前 mock 用于验证流程和分析框架 |

## 简历写法

搭建 AI 简历优化助手 Demo，支持简历文本/文件上传、JD 关键词解析、结构化优化建议生成与建议采纳反馈；设计 users、resume_tasks、events、suggestions 等核心数据表，基于模拟产品日志完成漏斗转化、Prompt A/B 实验、用户分群和 AI 建议质量评估，定位“采纳建议至导出报告”为主要流失环节，并提出导出入口前置、JD 关键词绑定和建议结构化展示等产品优化方案。
