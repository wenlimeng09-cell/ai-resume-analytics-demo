# 面试讲述稿

## 1. 项目背景

求职用户常常不知道如何根据不同 JD 修改简历，所以我做了一个可本地演示的 AI 简历优化助手 MVP。

这个项目不是完整商业产品，也不追踪真实投递和真实面试反馈。我的重点是验证简历优化流程中用户是否愿意采纳 AI 建议，以及如何用产品日志数据评估这个工具的效果。

## 2. 产品路径

用户进入产品后，可以上传简历文本或文件，粘贴目标岗位 JD。系统会基于规则模拟生成结构化建议，用户可以对每条建议选择采纳或不采纳，最后导出优化建议报告。

完整路径是：

访问产品 -> 上传简历 -> 上传 JD -> 生成建议 -> 采纳建议 -> 导出报告。

## 3. 数据设计

我没有只按用户维度做分析，而是新增了 `resume_tasks` 表。

原因是一个用户可能针对多个 JD 多次优化简历，如果只用 `user_id`，会混淆不同岗位和不同优化任务的效果。

核心表包括：

- `users`：用户信息；
- `resume_tasks`：简历优化任务；
- `events`：行为日志；
- `suggestions`：建议明细；
- `feedback`：可选体验反馈。

## 4. 指标体系

我重点设计了五类指标：

- 北极星指标：有效优化任务完成率，也就是完成导出报告且至少采纳 1 条建议的任务数 / 生成建议的任务数；
- 漏斗转化：看每个环节的用户数、转化率和流失率；
- 建议采纳：看建议曝光数、采纳数、采纳率和人均采纳建议数；
- Prompt A/B：对比通用建议型 Prompt 和 JD 匹配型 Prompt；
- 建议质量：从具体性、JD 匹配度和可执行性三个维度打分；
- 用户分群：对比应届生、实习生和转行用户。

## 5. 分析重点

我重点分析两个问题：

第一，哪些建议更容易被采纳。这里会看建议类型、简历模块、质量评分和采纳率之间的关系。

第二，JD 匹配型 Prompt 是否比通用建议型 Prompt 表现更好。这里会对比建议采纳率、报告导出率和平均建议质量评分，并用卡方检验展示分析方法。

第三，为什么用户采纳了建议但没有导出报告。这里我会用 exported 作为目标变量，按已采纳建议数、待处理建议数、匹配分提升、Prompt 组别和用户类型做分组导出率和相关性分析。

## 6. 产品结论

这个项目的结论不是“工具真实提升了面试率”，而是：

- 结构化建议更容易让用户理解和采纳；
- 展示 JD 关键词可以增强建议可信度；
- 一键采纳、复制改写文案、部分采纳可以降低操作门槛；
- 转行用户需要额外的可迁移能力识别和岗位差距分析；
- 有用 / 无用反馈可以持续优化建议生成逻辑。
