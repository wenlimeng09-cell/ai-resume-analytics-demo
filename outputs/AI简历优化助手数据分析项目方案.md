# AI 简历优化助手的数据分析与效果评估项目方案

## 一、项目定位

本项目模拟一款 AI 简历优化工具上线后的数据分析与产品效果评估。重点不是开发完整 AI 产品，而是站在数据分析 / 产品运营视角，回答一个核心问题：

> AI 简历优化助手是否有效提升了用户的求职效率？

适合包装为简历项目、作品集项目或面试案例。

## 二、业务背景

求职用户在投递岗位时，经常面临以下问题：

- 简历与 JD 匹配度不足；
- 不知道如何针对不同岗位调整简历；
- 修改简历耗时较长；
- 投递后反馈率低；
- AI 给出的建议可能泛泛而谈，用户未必愿意采纳。

因此，产品设计了一个 AI 简历优化助手，帮助用户上传简历、解析 JD、生成针对性修改建议，并辅助用户完成投递。

## 三、用户路径

用户完整路径如下：

1. 进入产品
2. 上传简历
3. 上传或粘贴 JD
4. AI 解析岗位要求
5. AI 生成修改建议
6. 用户采纳建议
7. 用户完成投递
8. 用户获得面试反馈

## 四、核心分析问题

围绕业务目标，可以拆解为以下问题：

1. 用户是否顺利完成核心使用路径？
2. 哪个环节流失最严重？
3. AI 建议是否提升了用户投递完成率？
4. 不同用户群体的使用效果是否存在差异？
5. 不同 Prompt 模板是否影响建议采纳率？
6. AI 建议质量是否影响用户后续投递和面试反馈？
7. 产品下一步应该优先优化哪里？

## 五、核心指标体系

### 1. 漏斗指标

| 漏斗环节 | 指标名称 | 计算方式 |
|---|---|---|
| 进入产品 | 访问用户数 | 进入产品页的去重用户数 |
| 上传简历 | 简历上传率 | 上传简历用户数 / 访问用户数 |
| 上传 JD | JD 上传率 | 上传 JD 用户数 / 上传简历用户数 |
| 生成建议 | 建议生成率 | 生成建议用户数 / 上传 JD 用户数 |
| 采纳建议 | 建议采纳率 | 采纳建议用户数 / 生成建议用户数 |
| 完成投递 | 投递完成率 | 完成投递用户数 / 采纳建议用户数 |
| 获得面试 | 面试反馈率 | 获得面试用户数 / 完成投递用户数 |

### 2. 留存指标

| 指标 | 含义 |
|---|---|
| 次日留存率 | 用户首次使用后第 2 天是否再次使用 |
| 7 日留存率 | 用户首次使用后第 7 天是否再次使用 |
| 周活跃用户数 | 7 天内至少使用一次产品的用户数 |
| 人均优化次数 | 用户平均生成简历优化建议的次数 |

### 3. 分群指标

可以按用户类型拆分：

- 应届生；
- 实习生；
- 转行用户；
- 有 1-3 年经验用户；
- 有 3 年以上经验用户。

重点比较：

- 各群体建议采纳率；
- 各群体投递完成率；
- 各群体面试反馈率；
- 各群体留存率。

### 4. A/B 实验指标

实验目标：验证不同 Prompt 模板是否能提升 AI 建议采纳率。

| 实验组 | Prompt 特征 | 观察指标 |
|---|---|---|
| A 组 | 通用建议型 Prompt | 建议采纳率、投递完成率 |
| B 组 | JD 匹配型 Prompt | 建议采纳率、投递完成率 |

核心指标：

- 建议采纳率；
- 投递完成率；
- 人均采纳建议数；
- 面试反馈率。

### 5. AI 建议质量指标

可以为每条 AI 建议打标签或评分：

| 质量维度 | 评分说明 |
|---|---|
| 具体性 | 建议是否具体到简历某一段、某一技能或某一项目 |
| JD 匹配度 | 建议是否对应 JD 中的岗位要求 |
| 可执行性 | 用户是否可以直接根据建议修改 |
| 表达质量 | 建议是否清晰、专业、没有空话 |
| 采纳结果 | 用户是否实际采纳该建议 |

可设计 1-5 分评分，并分析质量分与采纳率、投递率、面试反馈率之间的关系。

## 八、可能分析结论示例

示例结论可以这样写：

1. 整体漏斗中，“生成建议 → 采纳建议”环节转化率最低，说明用户对 AI 建议的信任度或可执行性感知不足。
2. B 组 JD 匹配型 Prompt 的建议采纳率高于 A 组，说明将建议明确绑定 JD 要求可以提升用户采纳意愿。
3. 转行用户的建议采纳率较高，但投递完成率较低，可能是因为其简历修改成本更高、岗位匹配信心不足。
4. 建议质量评分与采纳率呈正相关，其中“具体性”和“可执行性”对采纳率影响更明显。
5. 使用 AI 建议并完成投递的用户，其面试反馈率高于未使用建议用户，说明产品对求职效率有一定正向作用。

## 九、产品优化建议

### 1. 优化 AI 建议展示

将 AI 建议从泛泛描述改为“问题定位 + 修改建议 + 示例改写”三段式结构。

### 2. 增加 JD 匹配解释

在每条建议旁展示它对应的 JD 原文关键词，增强用户信任。

### 3. 降低采纳门槛

提供“一键应用建议”“部分采纳”“复制改写文案”等操作。

### 4. 针对转行用户提供专属路径

增加“可迁移能力识别”“经历重写模板”“目标岗位差距分析”等功能。

### 5. 建立建议质量反馈机制

允许用户对建议点击“有用 / 无用”，并用反馈数据持续优化 Prompt。

## 十、简历写法

可以在简历中写成：

**AI 简历优化助手数据分析与效果评估**

- 模拟 AI 简历优化产品从上传简历、解析 JD、生成建议、建议采纳到完成投递的完整用户路径，设计用户行为表、建议质量表和投递结果表，构建覆盖漏斗转化、留存、分群和 A/B 实验的指标体系。
- 使用 Python / SQL 对用户行为数据进行清洗与分析，定位“生成建议至采纳建议”为核心流失环节，并进一步从用户类型、Prompt 模板和建议质量维度拆解影响因素。
- 设计 A/B 实验评估不同 Prompt 模板对建议采纳率的影响，发现 JD 匹配型 Prompt 在建议采纳率和投递完成率上表现更优，并据此提出建议结构化展示、JD 关键词绑定和一键采纳等产品优化方案。

## 十一、面试讲述结构

面试时可以按这个顺序讲：

1. 我为什么选择这个项目；
2. 产品的核心用户路径是什么；
3. 我设计了哪些数据表和核心指标；
4. 我用漏斗分析发现了哪个关键问题；
5. 我如何通过分群和 A/B 实验进一步定位原因；
6. 我如何评估 AI 建议质量；
7. 我最终给出了哪些产品优化建议；
8. 如果真实上线，我会如何继续迭代。
